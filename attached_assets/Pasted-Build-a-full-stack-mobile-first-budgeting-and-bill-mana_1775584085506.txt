Build a full-stack mobile-first budgeting and bill management app inspired by Monarch and EveryDollar, but centered around a bill-first system with debt snowball automation and daily expense tracking.

TECH STACK:
- Frontend: React (Vite), mobile-first UI
- Backend: Python (FastAPI)
- Database: SQLite (use SQLAlchemy ORM)
- Charts: Recharts
- State: React Context or Zustand
- Styling: TailwindCSS (dark mode default like UI mockups)

--------------------------------------------------
CORE SYSTEM ARCHITECTURE
--------------------------------------------------

The system must follow this structure:

1. Bills (MASTER DATA)
2. Monthly Instances (GENERATED DATA)
3. Payments + Snowball Engine (LOGIC LAYER)
4. Daily Transactions (FLEXIBLE TRACKING)
5. Dashboard (AGGREGATION LAYER)

DO NOT duplicate data unnecessarily. Normalize properly.

--------------------------------------------------
1. BILLS TABLE (MASTER SOURCE)
--------------------------------------------------

Fields:
- id
- name
- amount
- category (Housing, Utilities, Debt, etc.)
- priority (1 = highest)
- is_debt (boolean)
- balance (for debts)
- interest_rate (optional)
- due_day (1–31)
- is_recurring (default true)
- created_at

Features:
- Full CRUD (add, edit, delete)
- Inline editing in UI
- Category tagging
- Priority sorting

--------------------------------------------------

DO NOT store static monthly copies unless needed.

Generate monthly data dynamically:

  - Pull all recurring bills
  - Create virtual instances

Optional table (if persistence needed):
MonthlyBill:
- id
- bill_id
- month
- year
- paid_amount
- remaining_amount
- status

--------------------------------------------------
3. PAYMENT + SNOWBALL ENGINE (CRITICAL)
--------------------------------------------------

Implement full debt snowball logic:

INPUT:
- Extra payment amount

PROCESS:
1. Sort debts by:
   - balance (ascending) → PRIMARY (snowball)
   - priority → SECONDARY

2. Apply minimum payments to all debts

3. Apply extra funds to smallest balance debt

4. When a debt is paid off:
   - Rollover its payment into next smallest debt
   - Continue until funds exhausted

OUTPUT:
- Updated balances
- Updated paid/remaining values
- Snowball progression visualization

Must support:
- Toggle between:
  - Snowball (smallest first)
  - Avalanche (highest interest first)
