The app must function like a hybrid of Monarch and EveryDollar but centered around a bill-first system with debt snowball automation and flexible monthly adjustments.

2. MONTHLY OVERRIDES (CRITICAL FEATURE)
--------------------------------------------------

Create a table to allow real-world variation:

MonthlyOverride:
- id
- bill_id
- month
- year
- custom_amount (nullable)
- paid_amount (default 0)

RULES:
- If override exists → use custom_amount
- Else → use default_amount

This allows:
✔ Bills that change month-to-month
✔ Partial payments
✔ Real-life flexibility

--------------------------------------------------
5. SNOWBALL ENGINE (ADVANCED)
--------------------------------------------------

Only applies to bills where is_debt = true

Process:
1. Sort debts:
   - Snowball → lowest balance first
   - Avalanche → highest interest first

2. Apply minimum payments

3. Apply extra funds:
   - Fully pay smallest debt
   - Roll freed payment into next debt
   - Continue cascade

Update:
- balances
- paid amounts
- monthly state

--------------------------------------------------
6. PAYMENT TRACKING LOGIC
--------------------------------------------------

For each month:

Paid = SUM(paid_amount)
Remaining = SUM(amount - paid_amount)

Persist payments in MonthlyOverride
